两个 source Handle: true / false，带标签

参考 Dify 的做法

Dify 的 IfElseNode 有:

  • 多个 case (IF / ELIF / ELSE)
  • 每个 case 有独立 sourceHandle
  • Handle 上有标签显示 "IF" / "ELSE"

mini-dify 最简版只需要:

  • 1 个条件表达式 → true / false
  • 2 个输出 Handle (true + false)

2. Edge sourceHandle (修改 WorkflowCanvas.tsx)

// onConnect 中保存 handle 信息
const onConnect = useCallback((conn: Connection) => {
  // 1. 通过 store.onConnect 保存到 Zustand (已有 sourceHandle/targetHandle 字段)
  store.onConnect({
    ...conn,
    sourceHandle: conn.sourceHandle ?? undefined,
    targetHandle: conn.targetHandle ?? undefined,
  });
  // 2. 同步到 ReactFlow 展示
  setRfEdges((eds) => [
    ...eds,
    { ...conn, id: `edge-${Date.now()}` } as Edge
  ]);
}, [store, setRfEdges]);

4. 变量传递语义 (修改 base.node.ts)

当前 __last_output 固定取最后 LLM 输出，分支场景会出问题。

改进方案
// Step 1: 在 NodeConfig 的 data 中支持 inputs 声明
// 例: data.inputs = { prompt: "{{llm-1.text}}", context: "{{start.query}}" }

// Step 2: BaseNode.getInputs() 解析 inputs 映射
protected getInputs(): Record<string, unknown> {
  const inputsConfig = this.config.data.inputs as Record<string, string> | undefined;
  if (!inputsConfig) return {};
  const result: Record<string, unknown> = {};
  for (const [key, template] of Object.entries(inputsConfig)) {
    result[key] = this.resolveTemplate(template);  // 已有方法
  }
  return result;
}

// Step 3: LLMNode 从 getInputs() 取值，不再依赖 __last_output
const inputs = this.getInputs();
const userPrompt = inputs["prompt"] ?? "Hello";
