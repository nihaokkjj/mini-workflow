后端设计: IfElseNode + 分支路由

核心问题：引擎如何根据条件选择 true/false 路径？

问题分析

当前 graph-engine.ts 做拓扑排序后按序执行所有节点，无法跳过分支。需要两个改动：

  1. IfElseNode 执行后，在 VariablePool 中标记 branch = "true" | "false"
  2. 引擎根据每条边的 sourceHandle 决定是否走这条边

设计: IfElseNode

新文件: backend/src/core/nodes/if-else.node.ts

IfElseNode.run() 伪代码
// 1. 读取条件表达式，如 "{{llm-1.text}} > 0.5"
const rawCondition = data.condition as string;
const resolved = this.resolveTemplate(rawCondition);

// 2. 安全求值: 用 Function 构造, 只允许简单比较
const result = this.evaluateCondition(resolved);
// 例: parseFloat("0.8") > parseFloat("0.5") → true

// 3. 输出分支标记
this.pool.setNodeOutput(nodeId, {
  result: result,
  branch: result ? "true" : "false",
  condition: resolved,
});

设计: 引擎分支路由

修改: graph-engine.ts 中的 run() 方法

分支选择逻辑 (在 run 循环内部)
// 执行节点后, 获取它的输出
const nodeOutputs = pool.getNodeOutput(nodeId);

// 构建应该执行的下游节点集合
const activeTargets = new Set<string>();
for (const edge of graph.edges) {
  if (edge.source !== nodeId) continue;

  // 如果 source 节点有 branch 标记, 只匹配对应 handle
  if (nodeOutputs?.branch) {
    if (edge.sourceHandle === nodeOutputs.branch) {
      activeTargets.add(edge.target);
    }
    // 不匹配的 branch → 跳过这个 target
  } else {
    // 普通节点 → 所有下游都执行
    activeTargets.add(edge.target);
  }
}

// 不活跃的 target → 从拓扑顺序中移除
for (const nid of remainingOrder) {
  if (!activeTargets.has(nid) && nid !== nodeId) {
    // 标记为跳过
    yield { event: "node_skipped", nodeId: nid, ... };
  }
}

条件求值安全性

不用 eval()，用正则匹配 + 简单比较：

// 安全的条件求值: 只支持 数字/字符串 的简单比较
private evaluateCondition(expr: string): boolean {
  // 模式: left op right  例: "0.8 > 0.5", "\"hello\" == \"hello\""
  const match = expr.match(/^(.+?)\s*(==|!=|>=|<=|>|<|contains)\s*(.+)$/);
  if (!match) throw new Error("Invalid condition: " + expr);

  const [, left, op, right] = match;
  const lv = isNaN(+left) ? left.trim() : +left;
  const rv = isNaN(+right) ? right.trim() : +right;

  switch (op) {
    case ">": return lv > rv;
    case "<": return lv < rv;
    case "==": return lv == rv;
    case "!=": return lv != rv;
    // ... 其他操作符
    default: throw new Error("Unknown operator: " + op);
  }
}
