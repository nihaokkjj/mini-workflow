validateGraph() — 在 run() 之前调用
private validate(): string | null {
  const startNodes = nodes.filter(n => n.type === "start");
  if (startNodes.length !== 1)
    return "Workflow must have exactly 1 Start node";

  const endNodes = nodes.filter(n => n.type === "end");
  if (endNodes.length === 0)
    return "Workflow must have at least 1 End node";

  // 孤立节点检查: 非 Start 节点入度 ≥ 1
  const hasIncoming = new Set(edges.map(e => e.target));
  for (const n of nodes) {
    if (n.type === "start") continue;
    if (!hasIncoming.has(n.id))
      return `Node ${n.id} has no incoming edge`;
  }

  // 节点类型合法性: 所有节点都有注册的构造器
  for (const n of nodes) {
    if (!NodeFactory.has(n.type))
      return `Unknown node type: ${n.type}`;
  }

  return null; // 验证通过
}

实现方式

推荐: 命令式渲染函数

  • 一个 map: NodeType → renderForm(nodeData, onChange)
  • 不需要为每种节点建独立组件文件
  • 和项目当前规模匹配，避免过度工程化

备选: 独立 Panel 组件 (Dify 方式)

  • 每种节点一个 panel.tsx
  • 适合复杂配置场景
  • 当前阶段过度设计
